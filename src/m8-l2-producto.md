## Evaluación

Antes de entregar un proyecto, es importante realizar una revisión general del producto.

### Checklist de evaluación

* [ ] ¿Cumple con el objetivo definido al inicio del proyecto?
* [ ] ¿Todas las funcionalidades principales operan correctamente?
* [ ] ¿La navegación es clara y fácil de utilizar?
* [ ] ¿La interfaz es consistente y comprensible?
* [ ] ¿Existen errores visibles durante el uso?
* [ ] ¿Hay código, archivos o recursos innecesarios?

> La evaluación permite detectar problemas antes de que el proyecto llegue a usuarios o clientes.

## Mejora

Una vez detectados los puntos débiles, es momento de optimizar el producto.

### Checklist de mejora

* [ ] Simplificar funcionalidades innecesarias.
* [ ] Mejorar mensajes de error y validaciones.
* [ ] Refactorizar o limpiar código repetido.
* [ ] Optimizar estructura de carpetas y archivos.
* [ ] Mejorar rendimiento cuando sea necesario.
* [ ] Actualizar la documentación.

> Un buen producto no solo funciona, también es fácil de mantener y utilizar.

## Feedback

El feedback permite obtener una visión externa del proyecto.

### Fuentes de feedback

* [ ] Compañeros de trabajo o estudio.
* [ ] Docentes o mentores.
* [ ] Usuarios de prueba.
* [ ] Clientes o usuarios finales.

### Aspectos a consultar

* [ ] Facilidad de uso.
* [ ] Diseño visual.
* [ ] Claridad de la información.
* [ ] Funcionamiento general.
* [ ] Posibles mejoras o nuevas funcionalidades.

> Muchas veces los usuarios detectan problemas que el desarrollador no observa durante el desarrollo.

## Ajustes

Luego de recopilar feedback, se realizan las correcciones finales.

### Checklist de ajustes

#### Interfaz y experiencia

* [ ] Coherencia visual.
* [ ] Navegación correcta.
* [ ] Diseño adaptable (responsive).
* [ ] Textos revisados y sin errores.

#### Proyecto

* [ ] Estructura de carpetas ordenada.
* [ ] Archivos innecesarios eliminados.
* [ ] Dependencias instaladas correctamente.
* [ ] Variables de entorno configuradas.

#### Funcionamiento

* [ ] Enlaces funcionando.
* [ ] Formularios funcionando.
* [ ] APIs respondiendo correctamente.
* [ ] Sin errores en consola.

> Los ajustes finales preparan el proyecto para su publicación o entrega.

## Checklist de entrega

Antes de cerrar el proyecto, verificar:

### Proyecto

* [x] Versión final completada.
* [x] Funcionalidades terminadas.
* [x] Errores críticos corregidos.

### Repositorio

* [ ] Código subido a GitHub.
* [ ] Commits organizados.
* [ ] Archivos innecesarios eliminados.

### Documentación

* [ ] README actualizado.
* [ ] Tecnologías documentadas.
* [ ] Instrucciones de instalación incluidas.

### Despliegue

* [ ] Aplicación publicada.
* [ ] URL funcionando correctamente.
* [ ] Proyecto accesible para revisión.

### Presentación

* [ ] Capturas de pantalla preparadas.
* [ ] Portafolio actualizado.
* [ ] Información de contacto disponible.

> Un proyecto bien entregado refleja profesionalismo, organización y buenas prácticas de desarrollo.
