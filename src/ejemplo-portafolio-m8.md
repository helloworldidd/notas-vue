## 1. Completar la aplicación

La aplicación debe incluir, como mínimo:

* Home con listado de lugares.
* Vista de detalle con pronóstico.
* Consumo de una API de clima.
* Manejo de estados (cargando y error).
* Vuex para el manejo del estado.
* Estadísticas y al menos una alerta meteorológica.

---

## 2. Revisar y mejorar

Antes de entregar verifica que:

* No existan errores visibles.
* La navegación funcione correctamente.
* El código esté ordenado.
* La interfaz sea clara y consistente.

---

## 3. Crear el README

Incluye, como mínimo:

* Enlace al repositorio.
* Requisitos del proyecto.
* Pasos para ejecutar la aplicación.
* Variables de entorno (si corresponde).
* Descripción de las principales funcionalidades.
* Capturas de pantalla (opcional).

---

## 4. Publicar el proyecto

Sube el proyecto a GitHub y entrega:

* Archivo `.zip` o enlace al repositorio.
* README completo.

---

## Ejemplo de proceso

**Paso 1** Finaliza todas las funcionalidades solicitadas.
**Paso 2** Prueba la aplicación y corrige errores.
**Paso 3** Sube el proyecto a GitHub.
**Paso 4** Completa el README con la información necesaria para ejecutar el proyecto.
**Paso 5** Comprime el proyecto (o comparte el repositorio) y realiza la entrega.

---

## Importante

La evaluación considera el correcto funcionamiento de la aplicación, la calidad del código, el uso de Vue, Vuex y la API, además de una documentación clara mediante el README y el repositorio en GitHub.
